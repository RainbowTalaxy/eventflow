```js
{

}
```

## 查看事件流程
GET /event/list

参数名|类型|必填|描述|默认值
:--|:--|:--|:--|:--
t_id    |number |是 |工作组 ID

返回值样例：
```js
{
    "list": [
        {
            "f_id": 101, // 事件流程 ID
            "completed": ["肥宅"], // 已完成人员
            "current": "宿霸", // 当前执行事件的人
            "inqueue": ["舍暴", "11"], // 待命的人
            "self": false, // 当前是否为自己（是否可提交“事件完成”申请）
            "isFounder": false, // 是否为工作组组长（是否可删除事件流程）
            "submitted": true // 当前执行事件的人是否提交了“事件完成”申请
        },
        {
            "f_id": 102,
            "completed": ["wjf", "pxx", "jyx", "hc", "cjw", "gsy"],
            "current": "csw",
            "inqueue": ["hdj", "xcj", "gq", "zrl", "yhl"],
            "self": true,
            "isFounder": false,
            "submitted": false
        }
    ]
}
```

## 添加事件流程
POST /event/add

参数名|类型|必填|描述|默认值
:--|:--|:--|:--|:--
t_id    |number |是 |工作组 ID
inqueue |string |是 |轮流参与的所有人员及顺序，比如 "肥宅, 宿霸, 舍暴, 11" （使用数组的 toString 方法）
f_type  |int    |是 |轮流的类型，顺序为 1 ，随机为 2 ，循环为 3

返回值样例：
```js
{
    "success": true, // 是否成功添加事件流程
    "info": "成功" // 具体信息
}
```
